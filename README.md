# Satellite Fault Detection — Single Layer Perceptron

A machine learning project that implements a Single Layer Perceptron (SLP)
from scratch using NumPy to classify satellite telemetry as nominal or fault.
Built as a portfolio project to demonstrate ML fundamentals and end-to-end
pipeline design.

## Dataset

**Phase 1 — Synthetic data**
Generated by `src/data_generator.py`. Simulates 6 satellite sensor channels:

| Sensor | Subsystem | Normal range | Fault signature |
|---|---|---|---|
| battery_voltage | Power | 24–28 V | 18–21 V |
| solar_current | Power | 1.5–3.5 A | 0–0.2 A |
| panel_temp | Thermal | -10 to 60 °C | 80–95 °C |
| onboard_temp | Thermal | 15–35 °C | 50–65 °C |
| attitude_error | ADCS | 0–0.5 deg | 2–5 deg |
| reaction_wheel_rpm | ADCS | 1000–5000 RPM | 0–100 RPM |

**Phase 2 — Real data**
ESA OPSSAT-AD dataset. Real telemetry from ESA's OPS-SAT CubeSat,
annotated by spacecraft operators.
Download: https://zenodo.org/records/12588359

---

## Model

The SLP is implemented from scratch in `src/perceptron_numpy.py` using
only NumPy. It uses the perceptron learning rule:

wᵢ ← wᵢ + η · (y − ŷ) · xᵢ

b  ← b  + η · (y − ŷ)

A sklearn Perceptron wrapper (`src/perceptron_sklearn.py`) is provided
for comparison. Both implementations share the same interface.

## Limitations and Next Steps

- The SLP can only learn linearly separable boundaries. Real satellite
  telemetry with overlapping fault and nominal readings will require
  a Multi-Layer Perceptron (MLP) or kernel-based method.
- Phase 2 will test the model on real ESA OPSSAT-AD telemetry where
  the classification challenge is realistic.
- A proper test set split will be added in Phase 2.
